Проект: Калькулятор на Ассемблере (Учебный)

Задача проекта:
Проект учебный, благодаря нему я понял, как работать со стеком, и также как преобразовать текст в число и наоборот.

Сам проект предоставляет функции калькулятора: сложение, вычитание, умножение и деление. Проект использует свои самописные функции, которые принимают аргументы и выполняют определенные действия в секции .text. Калькулятор также не упадет при ошибках, а отправит сообщение об ошибке, которая привела к сбою. Сообщения хранятся в секции .data.

Основные функции в проекте:

.trasform_args_in_digit — Принимает регистры:

    mov rdi = виртуальный адрес начала данных в конце стека.

    mov rsi = деление на что (10).

Задача: получить в определенных регистрах аргументы, которые отправил пользователь, и трансформировать их в число, используя алгоритм добавления места через умножение на 10. Как пример: у нас есть текст '1', '2'. Сперва мы преобразуем их в числа через вычитание 48. В алгоритме я сперва умножаю регистр, который равен 0, чтобы первое число не умножилось на 10, и потом уже добавляю первое число. Наш цикл повторяется: на второй итерации 1 становится 10, и этот нолик — это и есть место для второго числа. Так будет продолжаться, пока мы не дойдем до конца данных, которые нам отправили.

.urls_dispatcher_operation — Принимает регистры:

    rdi = первое число.

    rsi = второе число.

    rdx = операция.

    rax = результат.

Задача диспетчера: проверить, какую операцию от нас хочет пользователь. Мы сперва берем операцию из конца стека с помощью виртуального адреса и уже ставим его на определенный регистр для функции (dl = operation). И уже функция проверяет, если это операция, которую наш калькулятор может обработать. Если нет — наша функция прыгнет на обработку ошибки, а если да — тогда мы прыгнем к управляющим именно этой операцией, которую пользователь скинул, и она уже запустит свою функцию, которая сделает эту операцию и вернет нас обратно к следующей команде после нашего call .urls_dispatcher_operation.

.trasform_digits_in_text — Эта функция принимает регистры:

    rsi = деление на сколько (10).

    rdi = что надо поставить в буфер (r15).

    rbx = буфер, в который мы хотим поставить (remainder_storage).

    rcx = длина этого буфера, чтобы функция могла найти его конец (remainder_storage_len).

Функция .trasform_digits_in_text — это лишь конфигурация для другой функции (.handler_trasform_digits_in_text), которая уже использует эти регистры, чтобы преобразовать числа в текст.

Задача функции .handler_trasform_digits_in_text:
Взять число (rsi), разделить его на цифры и каждую цифру преобразовать в текст, добавляя 48 и сохраняя в буфер (remainder_storage) от конца до начала буфера (remainder_storage_len), чтобы не переворачивать буфер, а сразу иметь готовый буфер для системного вызова.

.system_call_for_write — Функция, которая принимает регистры rsi, rdi:

    rsi = мы предоставляем длину нашего буфера.

    rdi = мы предоставляем сам адрес этого буфера.

Функция берет эти аргументы, ставит в регистры для системного вызова write и заканчивает свою работу.

Основные обработки ошибок:

У нас есть четыре обработчика ошибок:

    Проверка аргументов (5 строка кода).

    Проверка на букву в аргументе (92 строка).

    Проверка операций (150 строка кода).

    Проверка на деление на 0 (185 строка кода).

У каждой проверки (кроме проверки на букву) есть свой конфигуратор, задача которого — подготовить регистры перед проверкой от диспетчера, который должен поставить в регистры системного вызова адрес ошибки:

    Конфигуратор аргументов (.error_args, 5 строка кода).

    Конфигуратор операций (.error_operation, 193 строка кода).

    Конфигуратор деления на 0 (.error_div_0, 185 строка кода).

Диспетчер Ошибок (.dispatcher_error):
Проверяет регистры и ставит адрес буфера ошибки из .data, и уже запускает .write_error, который выполняет системный вызов.
